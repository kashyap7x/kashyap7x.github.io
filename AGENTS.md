# Project Rules

This repository contains a static personal academic website generated by
`build.py`.

## Build Workflow

- Run `python build.py` after editing site data or generator code.
- The build reads `site_config.py`, `publication_list.bib`, `talk_list.bib`,
  `authors.json`, and Substack post metadata, then regenerates `index.html`,
  `publications.html`, and `talks.html`.
- The homepage fetches the latest posts for the configured Substack handle when
  reachable, writes display metadata to `substack_posts.json`, and falls back to
  that cache if the feed is unavailable.
- Python build dependencies are listed in `requirements.txt`.

## Editing Guidance

- Prefer editing the source data files, `site_config.py`, and `build.py`
  instead of manually editing generated HTML pages.
- Keep `build.py` user-independent where practical. Put profile/site
  customization in `site_config.py`.
- Update the `personal_data` configuration assembled in `get_personal_data()`
  in `site_config.py` for profile, bio, social links, footer, SEO metadata, and
  JSON-LD `Person` changes.
- Keep user-specific values centralized in `site_config.py`. Page titles,
  descriptions, profile image alt text, CV URL, organization, social/profile
  links, and highlighted self-author name should be derived from that config
  rather than hard-coded in `build.py`.
- Update `authors.json` when adding a linked author who is not already present.
  Author keys must exactly match the displayed BibTeX author name.
- Update `assets/site.css` for visual styling changes.
- Add publication thumbnails under `assets/img/publications/`.
- Add talk thumbnails under `assets/img/talks/`.
- Add PDFs, posters, and slides under `assets/pdf/`. Put publication posters
  under `assets/pdf/posters/` and talk slides under `assets/pdf/talks/`.

## Content Fields

Publications in `publication_list.bib` require `title`, `author`,
`booktitle`, `year`, `img`, and `html`. The generator uses `booktitle` as the
displayed venue even for `@Article` entries. Optional display/link fields are
`pdf`, `code`, `video`, `supp`, `poster`, `highlight`, `award`, and
`equal_contribution`.

Talks in `talk_list.bib` require `title`, `booktitle`, `year`, and `img`.
Optional display/link fields are `slides`, `video`, and `highlight`.

## Display Behavior

- Use `highlight = {true}` by convention for entries that should appear on the
  homepage; omit the field otherwise.
- Highlighted publications receive a light amber background on the full
  publications page.
- Publication `award` text is displayed after the title.
- `equal_contribution = {N}` marks the first N publication authors with
  asterisks.
